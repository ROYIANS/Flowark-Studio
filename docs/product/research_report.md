# Flowark 产品调研与规划报告 (Final)

> "AI 拉高了创作的下限，但也正在摧毁创作的上限。Flowark 的使命是让拥有独特品味的人，用 AI 放大他们的光芒。"

## 1. 核心理念与价值主张 (Core Philosophy)

### 1.1 时代的痛点：眼低手高与品味固化
在 AI 时代，技能（手）的门槛被极度拉平，但眼界（品味）却面临被算法平均化、平庸化的风险。
*   **现状**：用户依赖 AI 一键生成，导致内容千篇一律，丧失个人特质。
*   **后果**：创作者沦为 AI 的“快捷键操作员”，自身的创作能力和审美逐渐退化。

### 1.2 Flowark 的定位：品味放大器 (Taste Amplifier)
Flowark 不是一个简单的“效率工具”，而是一个**“以用户为主导的 AI 创作伴侣”**。
*   **核心差异**：
    *   传统 AI 工具：输入指令 -> 输出结果（替代用户思考）。
    *   Flowark：理解品味 -> 激发灵感 -> 辅助表达（放大用户思考）。
*   **价值主张**：
    *   **对于“看”**：通过高质量的灵感策展，提升用户的审美眼界。
    *   **对于“做”**：通过去框化的沉浸式编辑，让创作回归纯粹，AI 仅作为“手”来辅助。
    *   **对于“想”**：通过**《品味决策复盘报告》**，帮助用户反思和迭代，形成个人风格。

### 1.3 核心壁垒 (The "Secret Sauce")
我们认为，Flowark 最核心的竞争力在于构建了一套 **“人机共生创作流 (Human-AI Symbiosis Flow)”**：
*   **主导权归还**：AI 永远不抢夺用户的“方向盘”。在用户没有思路时，AI 提供选项而非答案；在用户有思路时，AI 默默退后仅做润色。
*   **品味建模**：系统能够通过用户的历史行为、选择偏好和人设档案，构建出用户的“品味模型”。AI 生成的每一个字、推荐的每一张图，都经过这个模型的“过滤”和“对齐”。

## 2. 竞品分析与市场机会 (Competitive Landscape)

### 2.1 市场现状坐标系

| 维度 | 效率工具箱 (Jasper, Copy.ai) | 通用编辑器 (Notion AI, 飞书) | 垂直平台工具 (抖音/小红书原生) | **Flowark** |
| :--- | :--- | :--- | :--- | :--- |
| **核心目标** | 速度、数量 (量产) | 通用性、文档管理 | 降低门槛、活跃度 | **质量、风格、个人成长** |
| **AI 角色** | 代工厂 (Factory) | 助手 (Assistant) | 引导员 (Guide) | **策展人 & 合作伙伴 (Curator & Partner)** |
| **内容风格** | 机械、同质化 | 平淡、白开水 | 平台风、跟风 | **鲜明、独特、有人味** |
| **用户体验** | 填表式 | 块状/文档式 | 碎片化 | **沉浸式、纸感、心流** |

### 2.2 为什么它们无法满足“有想法的创作者”？
*   **Jasper 等**：把创作者当成了“流量工人”，忽视了创作本身的乐趣和自我表达需求。
*   **Notion AI**：功能强大但过于理性冷淡，缺乏对“感性创作”和“特定人设”的深度支持。用户需要极高的 Prompt 技巧才能驾驭。
*   **平台工具**：被困在单一平台的算法里，无法跨平台沉淀个人的内容资产。

## 3. 目标用户画像 (Target Audience)

### 3.1 核心用户：有想法的表达者 (The Thoughtful Creator)
*   **特征**：
    *   **非小白**：知道自己要表达什么，有基本的审美判断力。
    *   **非专家**：可能缺乏专业的写作技巧、分镜知识或时间。
    *   **痛点**：有好的想法，但卡在“如何优雅地表达”或“如何适应平台规则”上；担心用 AI 会失去自己的风格。
    *   **需求**：需要工具帮我把 80 分的想法变成 95 分的作品，而不是帮我把 0 分变成 60 分。

### 3.2 典型场景
*   **场景 A**：一位生活美学博主，想写一篇关于“秋日居家”的笔记。她有画面感，但不知道怎么起一个抓人的标题，也不想写出营销号那种油腻的文案。
*   **场景 B**：一位职场导师，想把自己的经验拍成视频。他有干货，但不懂分镜和脚本结构，面对镜头容易紧张忘词。

## 4. 核心功能边界 (Feature Scope - MVP)

### 4.1 做什么 (Must Haves)
1.  **人设档案系统 (Persona Engine)**：
    *   定义语气、关键词、禁忌词。
    *   AI 基于人设的实时校验（“这句话不像你会说的”）。
2.  **浮光灵感 (Inspiration Curator)**：
    *   不是简单的热榜，而是经过筛选的“高品味”选题库。
    *   结构拆解：展示爆款内容的骨架，而非原文。
3.  **沉浸式编辑器 (Immersive Editor)**：
    *   极简 UI，去工具栏化。
    *   AI 交互：选中润色、指令面板、结构填充。
4.  **多模态脚本 (Multimodal Script)**：
    *   支持图文笔记和视频脚本两种视图的无缝切换。
5.  **品味决策复盘报告 (Taste Decision Report)**：
    *   **人设一致性诊断**：词汇图谱、语气偏离度分析。
    *   **内容感官节奏分析**：阅读呼吸图、情绪波动曲线。
    *   **数据与品味关联**：标题点击率归因、停留时间与节奏映射。
    *   **自我复盘**：创作日记（意图、创新点、失败点）。

### 4.2 不做什么 (Out of Scope)
1.  **不做全自动批量生成**：不支持“一键生成 100 篇”。
2.  **不做复杂的团队协作**：MVP 阶段专注于个人创作者。
3.  **不做多平台一键发布**：MVP 阶段专注于“创作”，发布通过复制或导出解决（避免早期陷入平台 API 的泥潭）。
4.  **不做通用的知识库**：我们不是笔记软件，是创作软件。

## 5. 产品生命力与增长逻辑 (Growth Loop)

### 5.1 核心增长飞轮：从“工具依赖”到“自我迭代”
Flowark 的粘性不依赖于无限生成，而是依赖于持续的个人成长。

1.  **看 (Inspiration Hub)**：看热点，更看审美和结构。
2.  **做 (Immersive Editor)**：以人设为锚点，高效产出。
3.  **想 (Decision Report)**：接收 AI 的品味诊断，将流量数据与创作决策关联起来，形成高质量的个人经验。

用户在 Flowark 待得越久，积累的**《品味决策复盘报告》越多，他的创作品味就越高，就越依赖 Flowark 提供的定制化诊断**，从而形成强大的产品壁垒。

### 5.2 关键指标 (North Star Metric)
*   **品味留存率 (Taste Retention)**：用户在查看"品味决策报告"后，下一次创作中采纳 AI 建议或改进创作习惯的比例。
*   **意图表达率 (Intent Expression Rate)**：用户在创作前主动记录"创作意图"的频率,以及在复盘时回看意图的频率。
*   **品味印记密度 (Taste Imprint Density)**：用户在创作过程中与系统进行"品味选择"类交互的次数(非生成类交互)。

## 6. 交互设计哲学：从"填表"到"对话" (Interaction Philosophy)

### 6.1 当前 AI 产品的交互困境

**问题诊断**：
*   **空白输入框焦虑**：面对一个大的输入框,用户不知道该说什么、说多少、怎么说。
*   **复杂表单疲劳**：过长的表单让用户在还没开始创作前就已经疲惫。
*   **想法内化困境**：即使用户脑子里有想法,也懒得打字表达,最终选择"让AI随便生成"。

**后果**：用户的真实想法没有沉淀到系统中,品味模型无法建立,产品沦为"一次性工具"。

### 6.2 Flowark 的交互策略：微交互 + 游戏化

**核心原则**：
1.  **降低单次表达成本**：不要一次性问太多,而是在创作流程中"见缝插针"地收集用户想法。
2.  **选择优于输入**：能让用户"选"的,就不要让用户"写"。
3.  **即时正反馈**：每一次表达,都要给用户即时的、可感知的价值回报。

## 7. 游戏化机制设计：品味成长体系 (Gamification System)

### 7.1 核心设计：品味印记系统 (Taste Imprint System)

**概念**：
每当用户在创作过程中做出一次"品味选择",系统就在用户的"品味图谱"上留下一个"印记"。随着印记的积累,用户的品味模型越来越清晰,AI 的辅助也越来越精准。

**印记的获取方式** (非创作行为,而是思考行为)：

| 触发场景 | 交互设计 | 获得印记 | 价值回报 |
| :--- | :--- | :--- | :--- |
| **看灵感时** | 看到灵感卡片,出现"❤️ 这个打动我"和"💡 这个有意思"两个快捷按钮 | +1 品味印记 | AI 记录你的偏好,下次推荐更精准 |
| **创作前** | 弹出"一句话说说你的想法"(限30字,有预设模板可选) | +2 品味印记 | 这句话会出现在复盘报告中,作为对比 |
| **写作时** | 选中一段文字,AI提示"这段的情绪是【焦虑】,符合你的预期吗?" 用户点击✓或✗ | +1 品味印记 | AI 校准对你的情绪理解 |
| **标题选择** | AI 生成3个标题,但不是直接用,而是让用户"投票":最符合我的是? | +1 品味印记 | 系统学习你的标题偏好 |
| **复盘时** | 看完数据后,系统问"这次的数据,符合你的预期吗?如果重来,你会调整什么?" 提供快捷选项 | +3 品味印记 | 将经验沉淀为可复用的规则 |

**关键设计**：
*   **不打断心流**：这些交互都是"可跳过"的,但跳过就没有印记。
*   **不是任务**：没有"每日必做",而是自然融入创作流程。
*   **即时可见**：每次获得印记,右上角有微动画 +1,点击可查看"品味图谱"。

---

### 7.2 品味图谱 (Taste Map)

**视觉设计**：
类似星座图,用温暖的手绘风格展示用户的品味分布:

```
        情绪共鸣
           ●
          /|\
         / | \
    故事性 ●---● 结构化
         \ | /
          \|/
           ●
        视觉美学
```

**动态演化**：
*   初期,图谱上只有几个点,很稀疏
*   随着印记增多,图谱逐渐丰满,每个维度有评分
*   当某个维度达到"精通"(100印记),解锁特殊标识,如"情绪大师 🎭"

**功能价值**：
*   **不是虚荣指标**：图谱直接影响AI推荐和生成质量
*   **成长可视化**：用户可以回看"3个月前的我,品味图谱长这样"
*   **社交货币**：未来可以生成精美的"品味卡片"分享(不带数据,只有风格)

---

### 7.3 成就系统：不是KPI,而是里程碑 (Achievement System)

**设计原则**：
*   ❌ 不设置"连续创作7天"这种强迫性任务
*   ✅ 只记录"品味突破时刻"

**成就类型**：

#### (1) 探索成就
*   **"风格冒险家"**：尝试了3种完全不同的写作结构
*   **"跨界先锋"**：在同一人设下创作了图文和视频两种形式
*   **"灵感猎手"**：收藏了10个不同类型的灵感

#### (2) 品味成就
*   **"一致性守护者"**：连续5篇内容的人设一致性评分>90%
*   **"节奏大师"**：创作了一篇"阅读心跳曲线"极其平滑的内容
*   **"标题诗人"**：你的标题平均点击率超过人设均值30%

#### (3) 成长成就
*   **"自我突破"**：某项数据相比上月提升50%,且你在复盘中明确指出了原因
*   **"品味沉淀"**：撰写了10条"创作日记"
*   **"百日筑基"**：品味图谱的5个维度都达到了20印记

**视觉呈现**：
*   不是弹窗,而是在"品味图谱"页面的下方,用极简的徽章展示
*   每个成就都有一段"手写体"的注解,如："你在用自己的方式,建立独特的叙事节奏"

## 8. 微交互设计：让表达变得"顺手" (Micro-Interaction Design)

### 8.1 创作前：意图捕捉的低门槛设计

**场景**：用户从灵感库点击"创作"

**传统方式** (❌)：
```
弹出对话框:
"请描述你的创作思路和目标受众..."
(大段输入框)
```

**Flowark 方式** (✅)：
```
┌─────────────────────────────────────────────┐
│  开始前,花 3 秒说说你的想法 👇             │
│                                              │
│  我想写/拍: (选一个)                        │
│  ○ 我自己的真实经历                         │
│  ○ 我观察到的现象                           │
│  ○ 我想分享的方法论                         │
│  ○ (自定义)                                 │
│                                              │
│  这次我想试试: (选一个或跳过)               │
│  □ 用故事开头  □ 用数据开头  □ 用提问开头  │
│                                              │
│  [跳过] [3秒说完了,开始创作]               │
└─────────────────────────────────────────────┘
```

**设计要点**：
*   **预设选项为主**：90%的想法都能用选项表达
*   **时间暗示**："3秒"降低心理负担
*   **可跳过**：不强制,但强调价值

---

### 8.2 创作中：对话式选择代替生成

**场景**：用户在编辑器中卡住了,不知道下一段写什么

**传统方式** (❌)：
*   AI直接续写一大段
*   或者出现输入框:"告诉AI你想要什么内容"

**Flowark 方式** (✅)：
```
(在光标位置出现轻量级悬浮卡片)

┌────────────────────────────────┐
│ 我注意到你停下来了,需要帮忙吗?│
│                                 │
│ 接下来你想:                    │
│ ○ 补充一个具体例子              │
│ ○ 过渡到下一个观点              │
│ ○ 总结一下前面说的              │
│ ○ 我自己想,不用帮忙 (消失)     │
└────────────────────────────────┘
```

**用户选择后**：
*   不是直接生成,而是给"提示句":
  ```
  (选了"补充例子")

  AI: 那这里可以这样展开:
  "就拿_______(事件)来说,当时我_______(细节),
   结果_______(转折)..."

  (光标自动定位到第一个下划线,等用户填空)
  ```

**价值**：
*   用户在"填空"过程中,是在**主动思考和表达**
*   这个过程产生的内容,100%是用户的语言

---

### 8.3 创作后：反思的游戏化

**场景**：用户完成一篇内容,点击"完成"

**传统方式** (❌)：
*   直接跳转到列表
*   或者弹出"分享到..."

**Flowark 方式** (✅)：
```
┌─────────────────────────────────────────┐
│  🎉 完成了!在归档前,记录一下这次的感受: │
│                                          │
│  这次创作的感觉是: (点击一个)           │
│  😌 很顺畅  😓 有点卡  💡 有突破      │
│                                          │
│  如果重来一次,你会: (可选)              │
│  ○ 调整开头方式                          │
│  ○ 增加更多细节                          │
│  ○ 换一个叙事角度                        │
│  ○ 我觉得已经很好了                      │
│                                          │
│  [暂时不想说] [记录完成 +3 品味印记]    │
└─────────────────────────────────────────┘
```

**2周后**,用户再次登录,系统会提示:
```
"你上次说'这次创作有突破',现在这篇的数据出来了,
 要不要看看你的'突破'是不是真的有效?"
```

## 9. 品味决策复盘报告：三层架构深化 (Decision Report Architecture)

### 9.1 第一层：数据镜像 (What Happened)

**设计原则**：只展示数据差异,不做好坏评价

**界面设计**：
```
┌──────────────────────────────────────────┐
│  《断舍离那篇》vs 你的常规表现            │
│                                           │
│  停留时长:     45s    (你的平均: 38s)    │
│  完读率:       78%    (你的平均: 62%)    │
│  收藏率:       12%    (你的平均: 8%)     │
│  评论情绪:     73%正面 (你的平均: 65%)   │
│                                           │
│  [查看详细分析]                           │
└──────────────────────────────────────────┘
```

---

### 9.2 第二层：品味决策回溯 (Why It Happened)

**核心功能**：将"创作意图" + "关键选择" + "数据结果"三者关联

**界面设计**：
```
┌────────────────────────────────────────────────┐
│  你的创作决策 → 内容呈现 → 读者反馈            │
│                                                 │
│  [创作前,你说]                                 │
│  "我想从'关系断舍离'的角度切入,因为我相信     │
│   人比物更重要"                                 │
│                                                 │
│  [创作中,你选择了]                             │
│  • 开头方式: 痛点共鸣式 (而非场景描述)        │
│  • 关键转折: 在第3段插入了个人故事             │
│  • 结尾方式: 留白式提问 (而非行动号召)        │
│                                                 │
│  [AI 节奏分析]                                 │
│  你在第3段用了具体故事("扔掉那段消耗我的关系") │
│  情绪曲线在这里出现波峰 ↗️ ↗️↗️              │
│  读者在这里的停留时间最长(12s),评论集中在    │
│  "太真实了"、"我也是"                         │
│                                                 │
│  [关联结论]                                    │
│  你的意图"人比物重要"→ 你选择用"关系故事"    │
│  → 读者产生"情感共鸣"                         │
│  这个决策链条是有效的 ✓                        │
│                                                 │
│  要把这个经验沉淀为你的"个人公式"吗?          │
│  [加入品味图谱] [暂时不]                       │
└────────────────────────────────────────────────┘
```

**关键设计**：
*   不是AI评价"你这里写得好",而是展示"你的选择 → 产生了这样的效果"
*   将因果关系可视化,强化用户的决策自信

---

### 9.3 第三层：品味成长档案 (How I Evolved)

## 11. 防止游戏化变质的设计原则 (Anti-Gamification Corruption)

**警惕**：游戏化容易变成"为了徽章而创作",偏离本质

**Flowark 的防护机制**：

1.  **印记不能刷**：
    *   同一类型的选择,短期内重复不加分
    *   例如:连续5次都选"痛点开头",第5次开始不给印记

2.  **成就不能强求**：
    *   没有"每日任务"
    *   成就都是"水到渠成"的,无法通过刷量获得

3.  **数据不排名**：
    *   品味图谱不与其他用户对比
    *   不显示"超过了85%的用户"

4.  **价值始终可见**：
    *   每个印记、每个成就,都明确关联到"AI会如何更懂你"
    *   不是虚荣指标,而是功能解锁

## 16. 从[xgd失败品]的失败中学到的教训 (Lessons from Glinta's Failure)

### 16.1 [xgd失败品]失败的三大根因

#### 根因1: 定位错误 - 工具思维 vs 成长思维

| 维度 | [xgd失败品] (工具思维) | Flowark (成长思维) |
|:---|:---|:---|
| **产品定位** | AI工具箱 - 7个独立Agent | 品味成长伙伴 - 统一创作流 |
| **用户角色** | 流水线操作员 | 创作主导者 |
| **AI角色** | 替代用户决策 | 辅助用户表达 |
| **价值主张** | "AI帮你做"(效率) | "AI帮你成长"(品味) |
| **用户感受** | 依赖AI,离开后无能为力 | 在AI辅助下,品味持续提升 |

**致命问题**:
- "账号定位Agent"、"选题策划Agent"都在**替用户做决策**
- 用户没有参与思考,只是在"填表 → 等待 → 下载"
- **停止付费后,用户一无所获** → 没有留存理由

**Flowark的正确做法**:
```
❌ 不做: "AI建议你做美妆赛道"
✅ 要做: "你最近点赞的灵感都是XX类型,这可能是你的兴趣方向"

❌ 不做: "本周推荐选题: 职场避坑指南"
✅ 要做: "你说想写关系类话题,这个热点符合你的人设吗?"
```

---

#### 根因2: 交互灾难 - 流程割裂与表单地狱

**[xgd失败品]的用户旅程**:
```
步骤1: 打开"账号定位" → 填10项表单 → 等待 → 下载PDF
步骤2: 打开"选题策划" → 填8项表单 → 等待 → 下载Excel
步骤3: 打开"编导脚本" → 填6项表单 → 等待 → 复制文案
步骤4: 打开"封面设计" → 填5项表单 → 等待 → 下载图片
...
```

**三大交互问题**:
1. **认知负荷爆炸**: 7个Agent,7套不同的交互逻辑
2. **重复劳动**: 同样的人设信息,要在不同Agent反复填写
3. **流程断裂**: 从"选题"到"脚本"要跨越多个页面,思路被打断

**用户崩溃点**:
- "我只是想写个脚本,为什么要填这么多表?"
- "我昨天填过的人设信息,今天又要重新填?"
- "这不是创作,这是在做问卷调查"

**Flowark的正确做法**:
```
统一的线性流程:
灵感浏览 → [3秒意图] → 沉浸式编辑 → [即时复盘] → 品味图谱
    ↓          ↓            ↓            ↓          ↓
  快捷按钮   选择题      微交互提示    emoji选择   自动沉淀

核心原则:
• 能"选"的绝不让用户"写"
• 能"自动"的绝不让用户"填"
• 能"渐进"的绝不一次问完
```

---

#### 根因3: 商业模式的自负 - 功能堆砌 ≠ 付费理由

**[xgd失败品]的逻辑**:
```
7个Agent → 功能很全 → 订阅制¥99/月 → 用户应该付费
```

**用户的真实想法**:
```
• 用了3次"选题Agent" → 都是烂俗热点,我自己刷抖音也能看到
• 用了2次"脚本Agent" → 生成的不是我的风格,还要大改
• 为什么要付费? 我直接问ChatGPT不是一样?
→ 流失
```

**核心问题**:
- **没有不可替代性**: [xgd失败品]能做的,ChatGPT也能做
- **没有数据沉淀**: 用户在[xgd失败品]的行为,换个工具就清零
- **没有情感连接**: 用户对产品无感,只是"用完即走"的工具

**Flowark的正确做法**:
```
付费的理由不是"功能多",而是"舍不得走":

1. 数据壁垒:
   我的品味印记、意图记录、决策历史,只有Flowark有

2. 情感壁垒:
   我的品味时间轴、月度进化报告,是我的创作自传

3. 成长见证:
   我在Flowark用了半年,明显感觉自己的品味在提升

→ 当用户想"换工具"时:
  "但我在Flowark的成长记录是无价的,舍不得丢"
```

### 16.2 Flowark 的5条避坑铁律

#### 铁律1: 永远不要"替用户做决策"

| 场景 | ❌ [xgd失败品]式 | ✅ Flowark式 |
|:---|:---|:---|
| 账号定位 | "AI分析: 你适合做美妆赛道" | "你最近收藏的灵感都是生活类,这可能是你的兴趣方向" |
| 选题推荐 | "本周推荐: 职场新人避坑指南" | "你说想写职场话题,这个热点'如何拒绝加班'符合你的人设吗?" |
| 标题生成 | "最佳标题: XXX(直接用)" | "3个标题,你觉得哪个最符合你的风格?(投票)" |

**核心**: AI只提供选项和参考,最终决策权在用户,且每个决策都要记录

---

#### 铁律2: 永远不要割裂流程

**[xgd失败品]的问题**: 7个独立Agent,用户要跨页面跳转

**Flowark的解决**:
```
整个创作在一个"沉浸式编辑器"完成:

[灵感库] → [意图捕捉] → [编辑器] → [复盘] → [品味图谱]
   ↓           ↓           ↓         ↓         ↓
 心动标记    3秒选择    微交互    emoji    自动沉淀

设计原则:
• 线性流程: 用户不需思考"下一步打开哪个功能"
• 上下文连贯: 前一步的选择,自动带入下一步
• 渐进增强: 核心功能先呈现,高级功能自然解锁
```

---

#### 铁律3: 永远不要用"复杂表单"

**能用"选择"的,绝不用"输入"**:

```
❌ [xgd失败品]: "请描述你的目标受众(年龄/性别/兴趣)_______"
✅ Flowark: "你想写给谁? ○职场新人 ○宝妈 ○学生 ○自定义"

❌ [xgd失败品]: "请输入内容方向_______"
✅ Flowark: "我想写: ○我的经历 ○观察到的现象 ○方法论"
```

**当必须输入时**:
- 限30字 + 提供模板 + 时间暗示("3秒说说")

---

#### 铁律4: 永远不要做"一次性输出"

**[xgd失败品]的问题**: 填表 → AI生成 → 下载PDF → **系统不知道用户选了什么**

**Flowark的做法**: 每次交互都要在系统中留下痕迹

```
场景: 用户选择"痛点共鸣式开头"

[xgd失败品]: 生成开头文案 → 用户复制走 → 结束

Flowark:
1. 记录选择 (+1品味印记)
2. 强化品味图谱的"痛点共鸣"维度
3. 2周后复盘: "你用'痛点开头'的6篇中,4篇完读率>70%"
4. 月度报告: "你在持续强化'痛点共鸣'风格"

→ 用户的每个选择,都在喂养自己的品味模型
```

---

#### 铁律5: 永远不要"为了AI而AI"

**[xgd失败品]的问题**: 7个Agent,每个都要显示"AI能力",但没有统一价值主张

**Flowark的原则**: AI只在3个关键时刻出现

| 时刻 | AI角色 | 用户感知 |
|:---|:---|:---|
| 用户卡住时 | 提供选项/提示句 | "AI在帮我理清思路" |
| 用户完成时 | 分析节奏/情绪 | "AI在帮我复盘" |
| 用户迷茫时 | 回顾历史,提炼模式 | "AI在帮我理解自己" |

**其他时间,AI要隐身**:
- 不要每个按钮都标"AI驱动"
- 不要频繁弹出"AI正在思考..."
- **让用户感觉在创作,而非调教AI**

### 16.3 关键功能对比: 如何正确替代[xgd失败品]的Agent

| [xgd失败品]的Agent | 问题 | Flowark的替代方案 | 核心差异 |
|:---|:---|:---|:---|
| **账号定位Agent** | 填表→生成报告→看完就忘 | **人设档案**(持续进化) | 不是一次性报告,而是在每次创作中逐步完善 |
| **选题策划Agent** | 填表→列出10个选题→用户挑 | **浮光灵感**(品味策展) | 不是AI选题,而是AI根据品味推荐,用户标记偏好 |
| **编导脚本Agent** | 填表→生成全文→用户改 | **提示句填空**(主动表达) | 不是AI写,而是AI给骨架,用户填血肉 |
| **标题标签Agent** | 填表→生成标题→直接用 | **标题投票**(品味学习) | AI生成3个,用户投票,系统学习标题审美 |
| **数据分析Agent** | 展示图表和数字 | **品味决策回溯**(因果关联) | 不止展示数据,还关联意图、选择、结果,沉淀经验 |
| **封面设计Agent** | 填表→生成封面→下载 | **配图建议**(侧边栏辅助) | 不独立成页面,而是在编辑器侧边栏自然呈现 |
| **变现顾问Agent** | 填表→生成变现方案 | **(MVP阶段暂不做)** | 专注创作本身,变现是水到渠成的结果 |

问用户: 你更喜欢哪种?

#### 阶段1: 技术验证 (Technical Validation)
**目标**: 验证核心AI能力是否可行

**关键技术挑战**:

##### 1. 品味建模 (Taste Modeling)
**问题**: 如何根据用户的选择,构建"品味模型"?

**技术方案探索**:
```
方案A: 基于Embedding的相似度匹配
   • 将用户的每次选择(标题偏好、开头方式、情绪标记)转为向量
   • 构建用户的"品味向量"
   • 推荐时计算余弦相似度

方案B: 强化学习
   • 将推荐视为Multi-Armed Bandit问题
   • 用户的点击/跳过作为奖励信号
   • 逐步学习最优策略

方案C: 规则引擎 + 轻量级ML (MVP推荐)
   • 前期用规则(如: 用户选过3次"痛点开头"→标记为偏好)
   • 积累数据后,训练简单的分类器
```

**验证方式**:
- 用模拟数据测试推荐精度
- 目标: 10次交互后,推荐准确率>60%

---

##### 2. 情绪与节奏分析 (Emotion & Pacing Analysis)
**问题**: 如何分析文本的"情绪曲线"和"阅读节奏"?

**技术方案**:
```
情绪分析:
• 工具: Sentiment Analysis API (如百度/腾讯AI开放平台)
• 按段落分析情绪值 → 生成曲线图

节奏分析:
• 特征: 句长、段落长度、标点密度
• 算法: 简单的统计分析 + 可视化
```

**验证方式**:
- 人工标注10篇文章的情绪曲线
- 对比AI分析结果,准确率>70%即可

---

##### 3. 提示句生成 (Prompt Sentence Generation)
**问题**: 如何生成"带填空"的提示句,而非全文?

**技术方案**:
```
方法: Few-shot Prompting

Prompt示例:
"用户想补充一个例子,请生成一个带填空的提示句:
示例输出: '就拿___(事件)来说,当时我___(细节),结果___(转折)...'

用户的上下文: [前文内容]
用户选择: 补充例子

请生成提示句:"
```

**验证方式**:
- 测试20个场景,人工评估提示句质量
- 目标: 80%的提示句"自然且有引导性"

#### 阶段2: MVP开发 (Minimum Viable Product)
**目标**: 开发一个可交互的原型,验证核心体验

**MVP范围** (基于用户验证结果,暂定):

##### 核心流程 (Must Have)
1. **人设档案**(简化版)
   - 只问3个问题: 名字、领域、语气
   - 后续在创作中逐步完善

2. **沉浸式编辑器**
   - 极简界面(无工具栏)
   - 支持图文创作(视频脚本暂缓)

3. **意图捕捉**
   - 创作前的"3秒选择题"

4. **提示句辅助**
   - 用户卡住时,提供选项 + 填空式提示句

5. **品味印记**(轻量版)
   - 只记录选择,暂不做可视化图谱
   - 每次选择有"+1印记"的微动画

6. **基础复盘**
   - 创作后的emoji感受记录
   - 2周后的提醒

##### 暂不做 (V2再考虑)
- 浮光灵感库 (用固定的mock数据代替)
- 品味图谱可视化 (数据记录,但不展示)
- 月度进化报告 (需积累数据)
- 成就系统 (游戏化元素)

**技术栈建议**:
```
前端: React + TailwindCSS (参考现有prototype.tsx的风格)
后端: Node.js + Express (轻量)
数据库: PostgreSQL (方便存储结构化的品味数据)
AI: OpenAI API / 国内大模型API (通义千问/文心一言)
```

**开发周期**: 6-8周 (2个月)
